初級日語學習週記 Week 7：みんなの日本語 L3

Day 1: これをください (2)

去上課的時候，補習班人員（不知道怎麼稱呼她比較好）問我 USA 先生上課感覺怎麼樣？跟她講一下，跟我表示她會跟老師反應。

上課前 USA 先生跟我說，他上次聽到我說我的名字「誠（まこと Makoto）」是來自 H-Game，就去問他的朋友，知道了《School Days》，還知道 Nice Boat 的哏（笑）。

還給我看了他月讀（女僕咖啡廳）的集點卡，像是對我表示他也是同圈子一樣，雖然之前就有想過 USA 先生是不是御宅……
這還聊到秋葉原跟日本橋，提到他很喜歡日本橋（USA 先生是神戶人，比較接近大阪。），聊過才知道原來東京也有一座橋叫日本橋。

今天上課只有我一個人！跟我一樣每次都會來上課的同學竟然沒來！是被 USA 先生（還是我）嚇到了嗎？教室除了我外，只有 USA 先生、翻譯的學姊（可以稱作學姊吧？），明明是補習班，卻是 1 對 1 教學，雖然賺到了，但總覺得有些不好意思。

上課一開始是 L2 的聽力練習，很快結束後開始課本的 L3 單字與句型，因為內容有關價錢的表達，後面大部份時間在練習數字的唸法，日文數字唸起來滿饒口的。

之中提到 1/2（にぶんのいち）的時候，USA 先生提到一首歌也是這名字，稍微想一下就知道是那首了？我問了「Anisong ですか`。」，USA 先生跟學姊都很驚訝，學姊還說了「果然是おたく（御宅族）。」（笑）。

巧合的是唱這首歌的歌手也叫 Makoto（川本真琴 Makoto Kawamoto）。

Day 2: これをください (3)

在 App Store 上看到「大家的日本語初級篇」APP，超強大！裡面包含了課本內容、講解本內容、有聲 CD，還有充滿時代感的情境影片（？），而且才賣 120 元，不過當然不是官方授權的，不然官方的課本都不用賣了（光是課本就快 200 元了），如果不在意版權的話，是很好的教材；免費的體驗版內容只到 L3，剛好可以讓我複習。

USA 先生上課前送了我禮物；日本這樣送禮是很平常的事嗎？還是這是希望我這最後的學生不要跑掉？

今天還是只有我一個人上課，感覺這門課之後都只有我一個人了……突然覺得日語補習班經營起來也滿辛苦的。

在家裡有複習過的關係，一開始上課複習很快就進入狀況。這次主要是練習聽力，覺得有點難度，尤其是唸到價格的部份，跟不太上。

USA 先生感覺一直急著進入 L4，是因為我上次跟補習班提到進度有點慢的關係嗎？但 L3 課文還沒講啊！

最後還是在練習數字，在投影片上列了很多數字，要我一個一個唸出來，在家有稍微練習過，所以不是很難， USA 先生一直誇我是天才，真是太誇張了；不過位數長一點就沒辦法反應那麼快，學姊還說可以唸快一點。

USA 先生要我加油，之後介紹日本留學生（♀）給我認識……來臺灣還要認識死肥宅，真可憐啊……

題外話：WS27

WS27 在 1030 開放入場，根據過往經驗，準時去一定還要排隊入場，所以我 1100 才到，但那個時間隊伍還是排到歷史博物館，看到那人龍，我決定先去便利商店買罐水，再開始汗臭排隊之旅；過往 WS 明明就不是那麼多人的展啊！有 LoveLive! 跟高捷少女加持差真多，還是說入宅的人變多了？

排隊過程看到一位 LLer 的戰衣，超霸氣！可惜我在排隊，不然就過去請他跟我合照了。

一進到會場馬上就知道為什麼要排隊，動線根本大爆死了！出入口同一個地方，而且在表演舞臺旁邊，然後熱門社團跟商業攤又集中在入口旁邊，擠到根本沒辦法動！

即使包含 Lovelive! Only 場，攤數也沒有特別多，有點失望，最大的攤應該是矮人礦坑，後面一大塊都是桌遊，連 40K 都擺出來了，但人沒有相對多，可以算是這次的場外休息區（？）。大致逛過一輪後，想找 Nomads 的攤挑戰地城戰棋的活動，眼睛稍微巡視一週沒看到，還在想是不是我搞錯了，其實沒有展出？還上網確認了一下，結果是位在熱門攤位前面，被人潮擋住了，在那個地方不知道該說幸運，還是不幸？

第一天的舞臺表演活動主要是 LL Coser 的舞蹈演出，興趣沒有很大，很快就出來去喝貢茶，然後排高捷集點活動兌換（也是要排隊）。即使到了下午，入場還是在排隊，場內的人好像越來越多了……

還有一件小插曲是舞臺上在唱〈僕らのLIVE 君とのLIFE〉這首時，我旁邊有一對情侶，女的走前面，那首歌詞有句「大好き（喜歡）」，唱到那段時，女的轉身對著男朋友跟著唱「大好き」，然後發現我在旁邊看，表情變得有點尷尬；那瞬間覺得這世界很美好，只是與我無緣（眼神死）。

拿到禮物跟抽獎獎品的瞬間，別人看我的眼神一定是死掉的，我後悔為什麼要跑高捷的十二個點……是我太晚來換了嗎？早知道一開始應該先排兌換，而不是排入場了！看到有人抽獎是抽到高捷少女簽繪，真讓人羨慕；六個章換到的是扇子（背面是靈夢），十二個章抽獎抽到的是御守……

第二天我中午才到，主辦單位因為有昨天的前車之鑑，會場的出入口換了地方，而且第二天的人數明顯少了不少，完全不用排隊，跟昨天差很多。

逛過一輪後到昨天注意的社團，買他們的商品，最後一邊逛一邊等待今天期待的舞台演出：KiRiKo 的二胡演奏

這次沒來還不知道，女性的 LLer 不少，Lovelive! 果然是不分男女、老少咸宜的優質作品（？）。國中生族群也不少，真讓人羨慕，我國中的時候，「御宅族」這名詞都還很難聽到，即使有動漫的興趣也是隱瞞起來，現實完全沒有同好，有點寂寞呢……

明明有 Lovelive! Only，結果買的都是原創物。這次買了 Krenz 的繪圖筆記，之後才察覺買錯了，想買的是畫冊才對；資料袋跟書籤是這次才知道的繪師「薪鹽」的作品，偏美式的簡約畫風，活潑、可愛，鮮明的用色看得很舒服，在大多數日系畫風創作的社團裡特別吸引我。

初級日語學習週記 Week 6：みんなの日本語 L1 - L3

我整個大爆炸的一週，爆炸威力大概等同第二次衝擊。

Day 1: 使徒襲來，停課一天！ほんのきもちです (4)、歷

上課一開始就來個 L1 - L2 聽寫小考，整個大爆死！L1 的單字超陌生，然後我的五十音原來還是沒很熟，根本除了 わたしは、です、ですか、ありません……這些跟句型有關的字外都寫不出來！。･ﾟ･(つд`ﾟ)･ﾟ･

L2 還好最近有讀，所以沒有那麼慘，不然我都懷疑我這幾週是學了什麼了。(´・ω・`)

小考考完再次複習完 L1 - L2，あいか老師教了暦（こよみ）。日文的日期真是有夠麻煩的，很多沒有照著規則來啊！

今天是あいか老師最後一天上課，準備跟我們道別前，老師就爆淚了……老師真感性。(´;ω;`)

還準備了禮物給我們，一邊流著淚一邊把禮物交給我的時候，對我說：「まことさん，你從一開始沒學過來學，現在進步那麼多，讓我很感動。」，聽完老師對我這麼說，讓我內心五味雜陳。祝あいか老師回到長崎也能保重。

一開始看到這包裝我一直在想會不會是肥皂？あいか老師也只說這在長崎很有名。

打開來才知道是看起來像餅乾的點心！捨不得吃啊……

Day 2: ほんのきもちです (5)、これをください (1)

今天很早就到教室，新老師在我到教室前就在裡面了，馬上給我一記猛虎原爆固定先問了我的姓名並寫在白板上。新老師的中文說得很差，對話起來有些雞同鴨講，沒多久負責翻譯的人進來了，原來還有問我日文綽號，還問我為什麼叫誠（まこと），我回答是某款 H-GAME 男主角的名字，讓我羞恥到爆炸。

上課一開始，老師先自我介紹，老師因為姓宇佐川（うさがわ Usagawa），以後週記就稱呼他美國河 USA 先生了。USA 先生的經歷滿豐富的，過去待過 HP 跟國際獵人頭公司，在中國大連與印度住過一段時間。還表示自己是 ASUS 的愛好者（ASUS 是把良率高的產品都賣到日本了嗎？），他身邊用的手機、平板跟筆電，還真的都是 ASUS。一直說臺灣人很聰明，所以學日語沒有問題……すみません，我是臺灣人中的智障。(´・ω・`)

USA 先生自我介紹完開始 L2 的練習……什麼？又是 L2 的練習？這樣的進度沒問題嗎？(((ﾟДﾟ;)))

あいか老師上課比較照著課本的進度，而 USA 先生上課會補充很多東西，會這樣可能是因為之前他上的都是高階會話課的關係？但 USA 先生的中文不好，他幾乎都是講日文，如果聽得懂能更清楚日本當地文化，但我都聽不懂啊！翻譯也不可能每句都即時翻譯。整堂課都在費神在揣測意思，整個大爆累！

USA 先生課程中提到，學習日語，唱歌是很好的辦法。問了我們喜歡什麼音樂，第一個先問我。

我：「anisong（動畫歌）」

USA：「那一首 anisong？」

我一時想不到那首，就回答了《God Knows…》，USA 先生馬上在 Google 搜尋曲名，不要這樣公開處刑我啊！XDDD

然後有點開心的發現作品場景在他的故鄉神戶（我已經忘記這件事了）。

中間休息後，開始 L3 的內容，可能前面太費神，有點不知道在上什麼……

今天上完課後，有種很強烈未來要靠自己努力的感覺（？）。

題外話

我家附近，從小學開始到現在經營有十年的漫畫出租店要結束營業了，現在的實體出租店結束營業都是時間早晚的問題，令人感受到物換星移的殘酷，雖然很少光顧，還是不禁感到一點哀傷。

我是個非常留戀於過去的時代，但我也很喜歡現代；網路的存在，讓一些事物化為時代的眼淚。卻也開創了各種的可能性，創作不再仰賴大公司，每個人都可以透過網路去募資創作自己的作品，再透過網路宣傳與販售；獨立電影、獨立音樂、獨立遊戲、獨立書籍……越來越多的獨立製作。

這件事把沉溺於過往的我又拉回了現實，讓我開始思考要怎麼繼續走向未來？自從退伍之後，我意識到自己是如此的平庸，甚至是愚笨、不知上進。從小到大想做的事很多：想創作遊戲、想創作一個很多人使用的軟體、腦中有至少四部想用小說、漫畫或遊戲呈現的故事、有一部天空與城市為主題的攝影集、想以社團身份參加同人創作展、學畫漫畫、學好日文跟英文了解那些我嚮往的文化、學笙、學薩克斯風、學吉他……跟國中生一樣，有很多很多想做的事。當開始發現自己要做到這些事是非常非常遙遠的時候，理想與現實的落差使我摔得粉身碎骨，只能懊悔過去沒有好好利用時間，但也沒有因此變得上進，繼續溺死在那過往的美好。

打算上完這期到七月中的日文，之後自己自學，然後找份工作。我不知道自己能做什麼，很懷疑自己的學經歷會不會被人看上眼，Github 上放的也都是垃圾，曾經考慮過到臺北找工作，發展性會比較高，不過臺北居大不易，總讓我有些怕怕的，當然這些擔心可能太自我感覺良好，畢竟也要找得到工作再說。最後可能還是要問問看、麻煩趙哥……或是我乾脆不寫程式了。

這社會上想做什麼事都要錢，我還是需要一份工作去得到錢，至少年底的時候可以買 PS4 與 SteamVR（認真）；有種說法是御宅族決不會殺人或自殺，因為他們還期待著下週動畫播出，跟期待的遊戲發售，這話不無道理，每次看到未來的新番預定表跟新遊戲發表（尤其這次 E3 的各種有生之年系列），都會又讓我有那一點「繼續活著真好」的感覺。所以 HL3 跟高分少女再連載呢？

我的好朋友唐宋今年大學畢業，星期一就要入伍當兵了，不管這是不是件好事，還是恭喜他畢業，也祝他當兵一年可以順遂平安。這樣未來一年誰來幫我 debug 日文啊！

初級日語學習週記 Week 5：みんなの日本語 L2

「か」跟「が」，以及「ね」跟「れ」，我很容易搞錯，以致上篇週記被唐宋糾正出很多錯誤，現在遇到那幾個容易搞錯的字，都會特別注意。

Day 1: ほんのきもちです (2)

因為跟《日本語 GOGOGO》的 L2 內容差不多，所以我有先在家裡複習與預習過一次。

一樣先從複習單字與句型開始，然後例文、會話與練習。

這次主要的三個句型：

  1. これはわたしの本です。（這是我的書。）
  2. これはわたしのです。（前面提過的話，可以省略受詞。）
  3. この本はわたしのです。（這本書是我的。算是第一句的倒裝？）

然後繼シャープペンシル後，又出現一個我超不會唸：
これからおせわになります。（今後還請您多多關照）

今天上課不太認真……

Day 1.5: デパートは何時から何時までですか

稍微看了一點《日本語 GOGOGO》的 L3，內容是時間的表示法，跟 L2 相較起來變得有點難，要開始進入學習曲線的低谷了嗎？

Day 2: ほんのきもちです (3)

練習、練習、再練習……這次有題聽力錯了，我不過是恍神了一下，我好傷心。

聽到一個令我悲痛的消息，あいか老師下星期要回長崎了，下星期一是她最後一次上課，我傷心到不想學日語了！。･ﾟ･(つд`ﾟ)･ﾟ･

下一位老師聽介紹說很像虎假面脫掉面具的樣子。(ﾟДﾟ)

以後唸錯就要吃一記猛虎原爆固定。

AngularJS Directives: ng-repeat

ng-repeat 是除了 ng-app 與 ng-controller 兩個一定會用到的 driectives 外，可能最常被用到的 driectives；可以用來迭代顯示陣列與物件中的資料。

迭代陣列

JS Bin on jsbin.com
ng-repeat 最常見的就是用來迭代陣列中的資料。

迭代物件

JS Bin on jsbin.com
也可以用來迭代物件。

除了使用物件之外，還有點不一樣的地方是，在 ng-repeat 裡引數也可以使用 (key, value) 的形式，用來顯示 key。另外在迭代物件時，它會根據 key 值進行排序；如果改用陣列，key 就會是陣列的 index。在我的書上有還有提到會根據英文字母與大小寫進行排序，但我實作出來是沒有效果的。

helper 變數

JS Bin on jsbin.com
ng-repeat 提供一些 helper 變數來方便開發。

$first、$middle、$last

輸出都是 boolean，就像變數名稱一樣，分別用來表示項目是第一項、中間項，還是最後項。

$index

該項目在陣列裡的牽引。要注意的是，如果迭代的是物件，不是根據物件的 key，而是排序後的 index。

$odd、$even

輸出都是 boolean，用來表示項目的牽引是基數項或偶數項。

追蹤

JS Bin on jsbin.com
追蹤是比較進階的用法，用來優化效能。ng-repaet 為了追蹤資料的新增、刪除與修改，會在 DOM 中加入 $$hashKey 屬性來當作識別。而從上面的 DEMO 來看，如果資料更新後，即使內容相同，還是會重新產生 DOM。

JS Bin on jsbin.com
在 ng-repeat 的引數加上 track by 就可以用我們定義的屬性當作識別，當資料內容相同時，不會另外產生 DOM，也不會新增 $$hashKey 屬性。這在資料來源來自資料庫，而且資料量很大（幾萬筆資料）的狀況，效益特別大。

跨容器

JS Bin on jsbin.com
ng-repeat 使用上也可以跨兩個容器（就只能兩個）。這個比較少用，通常需要 ng-repeat 跨兩個或兩個以上的容器，應該會選擇重新設計 HTML……

Facebook 與 Open Graph tags

每次看到 Tony 在 Facebook 上的文章連結，再看看我的，總覺得就是輸了一截；少了圖，標題跟簡述都醜醜的……技術力差了一大截，但這部份至少要相同！

Facebook 抓這些資訊，主要是根據 The Open Graph protocol 定義的 Open Graph tags，所以我只要加上去就可以了。

Open Graph tags 直接寫在 HTML meta 的屬性裡，其中 Facebook 上會用的是：og:title、og:description、og:image 等。

og:image

原本以為我用的 Hexo themes 本來沒有加上 Open Graph tags，結果我直接拿 og:image 當關鍵字搜尋：

<!-- themes\light\layout\_partial\head.ejs -->
...
<% if(page.cover) { %>
  <meta property="og:image" content="<%= page.cover %>" />
<% } else { %>
  <meta property="og:image" content="<%= config.cover %>"/>
<% } %>
...

原來早就定義好了，還會判斷是 page.cover（文章），還是 config.cover（全站）。我只要直接在 _config.yml 裡加上 cover 就好。

// _config.yml
...
cover: http://otakug.github.io/img/cover.jpg
...

og:title

og:title 也一樣已經定義好：

<!-- themes\light\layout\_partial\head.ejs -->
...
<% if (page.title){ %><meta property="og:title" content="<%= page.title %>"/><% } %>
...

稍微改一下，加上 config.title ，讓它能出現網站名稱：

<!-- themes\light\layout\_partial\head.ejs -->
...
<% if (page.title){ %><meta property="og:title" content="<%= page.title %> | <%= config.title %>"/><% } %>
...

og:description

og:description 就沒定義了，倒是有定義 meta 原本的 description，自己加上去就好：

<!-- themes\light\layout\_partial\head.ejs -->
...
<% if (page.description){ %>
<meta name="description" content="<%= page.description %>">
<% } else if (config.description){ %>
<meta name="description" content="<%= config.description %>">
<% } else if (page.excerpt){ %>
<meta name="description" content="<%= strip_html(page.excerpt).replace(/^\s*/, '').replace(/\s*$/, '') %>">
<% } else if (page.content){ %>
<meta name="description" content="<%= strip_html(page.content).replace(/^\s*/, '').replace(/\s*$/, '').substring(0, 150) %>">
<% } %>

<!-- 自己新增的 og:description -->
<% if (page.description){ %>
<meta property="og:description" content="<%= page.description %>">
<% } %>
...

那些變數預設也是都沒有定義；不過，Facebook 還是會擷取網站內容當作 description，這邊有需要再另外加就好。

Hexo 上能自動調整寬度的 YouTube 影片

前天晚上，Tony 丟給我一張截圖。

我的繪麻小天使滿出來了！

其實在很早之前，我就有發現這問題，但我實在懶得想辦法改 Hexo 的 source。即使 Tony 反應，我還是沒有想改的念頭。

昨天早上，Tony 又丟了東西給我：CSS-Tricks: Fluid Width Video

啊啊啊啊！他還幫我找到解法了啊！感恩 Tony，讚嘆 Tony！

好吧！現在沒有 Responsive Web Design，真的很遜，就來改吧！

CSS

照 CSS-Tricks 網頁上寫方法的來看，我只要在 iframe 外在包一個父元素，再改變它們的 CSS 就好了。

那就先找到 Hexo 的 CSS 下在那裡。Hexo 設計上的重複利用性很高，前端內容都是透過 themes 裡的 source 去動態產生，所以只要更動 themes，全部頁面就會一起更動。

很直覺就能找到，CSS 是寫在 /themes/light/source/css/（light 是我目前使用的 theme）這個路徑裡。

這路徑打開一看……奇怪，怎麼會是我沒看過的 .styl？說好的 .css 呢！

沒關係，先隨便找一個打開來看一下：

// /themes/light/source/css/_partial/commant.styl
#comment
  padding 20px
  background #fff
  box-shadow 1px 2px 3px #ddd
  margin-bottom 50px
  .title
    font-weight normal
    margin-bottom 15px

一看就懂，有點像 SASS，那我再把 CSS-Trick 提供的 CSS 改成同樣的規格加上去就好了。

在感覺上是全域樣板的 layout.styl 裡，在最後面加上：

// /themes/light/source/css/_base/layout.styl
.videoWrapper
  position relative
  padding-bottom 56.25%
  padding-top 25px
  height 0
  iframe
    position absolute
    top 0
    left 0
    width 100%
    height 100%

從這 CSS 來看，這是很常見的寫法。父元素的 possition 設為 relative，子元素設為 absolute，子元素就能以父元素為基準改變位置與大小。在這個例子裡是讓子元素的長寬設為 100%（基準是父元素），我們調整父元素的 padding 間接改變子元素的大小。

再把文章上的 iframe 用 div 包起來：

...
<div class="videoWrapper">
    <iframe width="560" height="315" src="https://www.youtube.com/embed/WUODSIxcMC8" frameborder="0" allowfullscreen></iframe>
</div>
...

最後用 Chrome 的開發者工具或直接用手機看一下：

YEAH！完成！

等等，好像那裡不對！在這篇文章之前，blog 只有四個影片，所以一個一個新增 div 可以很快，但假如我之前有一百個、上千個影片，不就加到崩潰痛哭？而且在每次在文章嵌入 YouTube ，都要自己新增一個 div，實在有點奇怪。我們需要一支程式來解決這個問題！

JavaScript

CSS-Tricks 那篇文章上面已經有提供用 jQuery 寫的一支自動調整影片寬高的程式，但直接照著寫，實在沒什麼意思。而且我只要簡單把每個 YouTube 的 iframe 各自用父元素包起來就好了。code 好多行好可怕，我有看到 code 太多行會死的病。

在 /themes/light/source/js/ 裡新增一個檔案，取名叫 fluidVideo.js。然後裡面的內容：

// /themes/light/source/js/fluidVideo.js
// 取得頁面中嵌入的的 YouTube 影片
var allVideos = $("iframe[src*='//www.youtube.com']")

// 把所有的 iframe 包一個 videoWrapper 容器（父元素）
$(allVideos).each(function() {
    $(this).wrap("<div class='videoWrapper'></div>")
});

作筆記

從以前在學校到現在就很不喜歡作筆記，除了覺得麻煩外，另一方面是覺得自己的字很醜，看得就討厭。比起手寫，當然是更喜歡數位筆記。自從上課後，發現我必須透過筆記去記錄，方便平常的複習，所以我開始摸索與嘗試筆記的模式。希望有人也可以給我建議。

一開始打算用一本小小的筆記本就搞定，但我實在沒辦法同時很專心作筆記跟聽課。我又再買了一本比較大的筆記本，小的筆記本還是上課時用，內容寫得很快很草，只有自己看得懂的程度，回到家再把小筆記本上的內容，重新整理到大筆記本上，在重新謄寫的過程還可以順便複習。

我還上網查了一下要怎麼在筆記上應用多種顏色，不過找不到一個我覺得適合的例子，所以我決定用自己的模式：
黑色－主要
藍色－強調
綠色－中文解釋或補充說明
紅色－特別重要（現在還沒出現過）
